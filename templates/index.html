Protein Reference Dataset

Protein Sequence Data

Orthology inference is most often based on molecular protein sequences. For a comparison of different orthology prediction methods, a common set of sequences must be established. Therefore, only identical proteins are mapped to each other.

To make comparisons of method easier, the orthology research community has agreed in 2009 to established a common reference proteom dataset. You can choose whether your orthology predictions should be assessed on the base of this reference dataset or on the proteom set of OMA.

Protein Sequence Data

If you decide to go for the OMA protein dataset, you are supposed to upload your protein sequences. Currently, we support Fasta sequence formats. It is expected that each label starts with a unique ID and the species is somehow encoded, e.g. SwissProt/UniProt species codes or ncbi taxonomic ids

Ortholog Relation Data

Ortholog Relation Data

Our tests assess orthology on the bases of protein pairs. Therefore, we ask our users to upload their prediction in a format from which we can extract pairwise relations in an unambiguous manner: We support

  • simple simple text file with two tab-separated columns of orthologous protein represented by their ids
  • orthoxml v0.3, which allows for nested orthologGroups and paralogGroups.

For both formats, we expect you to submit your predictions in a single file.

Text (pairwise orthologs) orthoXML v0.3

Example Datasets

  • A small dataset comprising 7 Bacteria genomes in the pairwise text format: BacExample.txt (273Kb)
  • Eukaryotic dataset with 9 genomes in pairwise text format: EukBBH.txt (5Mb)
  • Dataset of OMA Groups on reference proteomes in orthoxml format: OMAGroups.orthoxml (45Mb)
